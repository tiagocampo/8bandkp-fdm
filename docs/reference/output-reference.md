# Output File Reference

All output files are written to the `output/` directory (created automatically if absent). The code uses a consistent internal unit system throughout: **eV** for energy, **Angstrom** for length, and **1/Angstrom** for wave vectors.

Files are ASCII, space-delimited, and use Fortran `g14.6` formatting (14-character fields with 6 significant digits). Unless noted otherwise, files have no column headers in 1D formats; 2D grid files include `#`-comment headers and blank-line row separators for gnuplot `splot` compatibility.

---

## 1. Band Structure Output

### `output/eigenvalues.dat`

Eigenvalues at each k-point in the wave-vector sweep. Written by `bandStructure` and `gfactorCalculation`.

**Generated by:** All modes (`bandStructure` with `confinement=0`, `1`, or `2`; `gfactorCalculation` with any confinement).

**Format:** ASCII, one row per k-point. The number of eigenvalue columns equals `numcb + numvb`.

**Wire-mode header** (`confinement=2` only): one metadata line before the data:
```
# confinement: 2 nx: <nx> ny: <ny> dx: <dx> dy: <dy> shape: <shape>
```

**Data columns:**

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | `\|k\|` = `sqrt(kx^2 + ky^2 + kz^2)` | 1/Angstrom |
| 2 .. N+1 | Eigenvalues sorted ascending (valence then conduction) | eV |

**Interpretation:** For bulk (`confinement=0`), there are always 8 eigenvalues per k-point (2 conduction, 6 valence). For QW (`confinement=1`), there are `numcb + numvb` eigenvalues where `numcb = 2 * fdStep` and `numvb = 6 * fdStep`. For wire (`confinement=2`), the count equals the input `numcb + numvb` directly (not scaled by grid size).

## 2. Wavefunction Output

### `output/eigenfunctions_k_XXXXX_ev_YYYYY.dat`

Eigenfunction amplitudes for a specific eigenstate at a specific k-point. Zero-padded 5-digit indices: `k` is the k-point index (1-based), `ev` is the eigenvalue index (1-based, ascending energy).

**Generated by:** All modes. Written at the first, middle, and last k-points for bulk/QW. For wire mode, written at k=1, the midpoint, and the last k-point.

#### Bulk format (`confinement=0`)

8 values per row (no position column). The eigenvector is 8-dimensional (one component per band).

| Column | Meaning | Units |
|--------|---------|-------|
| 1..8 | `\|psi_band\|` for bands 1..8 | dimensionless (not normalized to 1) |

**Interpretation:** Band ordering: 1=HH, 2=LH, 3=LH, 4=SO, 5=split-off, 6=split-off, 7=CB, 8=CB. The squared modulus gives the band character.

#### Quantum well format (`confinement=1`)

One row per spatial grid point. Each eigenvector has `8 * fdStep` components; the file shows the 8 band components extracted at each z-position.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | z-coordinate | Angstrom |
| 2..9 | `\|psi_band(z)\|` for bands 1..8 at this z | dimensionless |

**Interpretation:** The probability density for a given band at position z is column^2. The total probability density is `sum(bands 1..8) of column^2`.

#### Wire format (`confinement=2`)

One row per grid point, with blank lines between y-rows (gnuplot `splot` format). Includes a header line with the eigenvalue.

```
# E = <eigenvalue>
x  y  |psi|^2
...
(blank line between y-rows)
```

| Column | Meaning | Units |
|--------|---------|-------|
| Header | Eigenvalue `E` | eV |
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | `\|psi(x,y)\|^2` summed over all 8 bands | 1/Angstrom^2 |

**Interpretation:** The probability density at each grid point, summed over all 8 band components. For a normalized eigenvector, `sum(|psi|^2 * dx * dy)` over all grid points equals 1.

## 3. Band Character Output

### `output/parts.dat`

Band decomposition (fractional character) for each eigenstate. One row per eigenstate.

**Generated by:** All modes. Written alongside eigenfunctions at the same k-points.

**Format:** 8 space-delimited values per row, no header.

| Column | Meaning | Units |
|--------|---------|-------|
| 1..8 | Fractional weight of bands 1..8 | dimensionless |

**Bulk calculation:** `parts(j, band) = |A(band, j)|^2` (squared modulus of the eigenvector component). These sum to 1 for each eigenstate.

**QW calculation:** `parts(j, band) = integral_z |psi_band(z)|^2 dz`, approximated as `sum(|psi_band(z)|^2) * dz`. These also sum to approximately 1.

**Wire calculation:** `parts(j, band) = sum_{grid} |psi_band(x,y)|^2 * dx * dy`. These sum to approximately 1.

**Interpretation:** Band ordering: 1=HH, 2=LH, 3=LH, 4=SO, 5=split-off, 6=split-off, 7=CB, 8=CB. A conduction band state has most weight in columns 7 and 8; a heavy-hole state has weight in column 1.

## 4. g-Factor Output

### `output/gfactor.dat`

Landau g-factor tensor eigenvalues along the three Cartesian directions. Written by `gfactorCalculation`.

**Generated by:** `gfactorCalculation` only, all confinement modes (`0`, `1`, `2`). Always computed at the Gamma point (k=0).

**Format:** Single row, 3 values, no header.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | g_x | dimensionless |
| 2 | g_y | dimensionless |
| 3 | g_z | dimensionless |

**Interpretation:** The g-factor in each direction is `2 * lambda_min` where `lambda_min` is the smallest eigenvalue of the 2x2 Zeeman tensor for that direction. Positive g means the spin splitting follows the free-electron sign convention. The free-electron value is g = 2.00231.

### `output/optical_transitions.dat`

Optical transition matrix elements and oscillator strengths between conduction and valence subbands. Written by `gfactorCalculation` in wire mode only.

**Generated by:** `gfactorCalculation` with `confinement=2`.

**Format:** One header line (`# CB VB dE(eV) |px|^2 |py|^2 |pz|^2 f_osc`), then one row per transition.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | CB subband index | dimensionless |
| 2 | VB subband index | dimensionless |
| 3 | Transition energy `dE` | eV |
| 4 | `\|<CB\|dH/dk_x\|VB>\|^2` (p_x momentum matrix element squared) | (eV * Angstrom)^2 |
| 5 | `\|<CB\|dH/dk_y\|VB>\|^2` (p_y) | (eV * Angstrom)^2 |
| 6 | `\|<CB\|dH/dk_z\|VB>\|^2` (p_z) | (eV * Angstrom)^2 |
| 7 | Oscillator strength `f = sum(\|p\|^2) / (hbar^2/(2m0) * dE)` | dimensionless |

## 5. Potential Profile Output

### `output/potential_profile.dat`

Band edge profile (material band offsets before any self-consistent correction).

**Generated by:**
- `bandStructure` with `confinement=1` (QW): written once at startup.
- `bandStructure` with `confinement=2` (wire): written after strain and SC corrections (if any), before the k-sweep.

#### QW format (`confinement=1`)

One row per z-grid point, no header.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | z-coordinate | Angstrom |
| 2 | E_V (valence band edge) | eV |
| 3 | E_V - Delta_SO (split-off valence band edge) | eV |
| 4 | E_C (conduction band edge) | eV |

#### Wire format (`confinement=2`)

2D grid with header and blank-line separators (gnuplot `splot` format).

```
# x(A) y(A) EV EV_DeltaSO EC
```

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | E_V | eV |
| 4 | E_V - Delta_SO | eV |
| 5 | E_C | eV |

**Interpretation:** The band gap at each point is `E_C - E_V` (column 5 minus column 3 for wire, column 4 minus column 2 for QW). Band offsets are set by the material parameter database; `EV` is the heavy/light hole edge, `EV - DeltaSO` is the split-off hole edge.

## 6. Self-Consistent Output

These files are written when `SC=1` is set in `input.cfg` and the self-consistent Schrodinger-Poisson loop converges.

### `output/sc_potential_profile.dat`

Band edge profile after self-consistent convergence. Has the same format as `potential_profile.dat` but reflects the potential shifted by the electrostatic solution.

**Generated by:** `bandStructure` with `confinement=1` (QW) or `confinement=2` (wire) when `SC=1`.

**QW format:** One row per z-grid point, no header.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | z-coordinate | Angstrom |
| 2 | E_V (self-consistent) | eV |
| 3 | E_V - Delta_SO (self-consistent) | eV |
| 4 | E_C (self-consistent) | eV |

**Wire format:** 2D grid with header `# x(A) y(A) EV EV_DeltaSO EC`, blank-line separators.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | E_V (self-consistent) | eV |
| 4 | E_V - Delta_SO (self-consistent) | eV |
| 5 | E_C (self-consistent) | eV |

**Interpretation:** Compare with `potential_profile.dat` to see the electrostatic shift. The difference is the Hartree potential `-e * phi(z)` applied to the band edges.

### `output/sc_phi.dat`

Electrostatic potential from the Poisson solver. Wire mode only.

**Generated by:** `bandStructure` with `confinement=2` and `SC=1`.

**Format:** 2D grid with header `# x(A) y(A) phi`, blank-line separators.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | phi (electrostatic potential) | eV |

**Interpretation:** The electrostatic potential solved from the Poisson equation with the charge density as source. Boundary conditions are set by `SC_bc` (Dirichlet-Dirichlet or Dirichlet-Neumann) with values `SC_bc_left` and `SC_bc_right`.

### `output/sc_charge.dat`

Electron and hole charge densities from the self-consistent solution. Wire mode only.

**Generated by:** `bandStructure` with `confinement=2` and `SC=1`.

**Format:** 2D grid with header `# x(A) y(A) n_e n_h`, blank-line separators.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | n_e (electron density) | 1/Angstrom^2 |
| 4 | n_h (hole density) | 1/Angstrom^2 |

**Interpretation:** Charge densities are computed from the occupied k.p eigenstates integrated over in-plane (k_x) wave vectors. The total charge is `sum(n_e - n_h) * dx * dy` over the grid.

## 7. Wire-Specific Output

### `output/strain.dat`

Strain tensor components on the 2D grid. Written when strain is enabled in wire mode.

**Generated by:** `bandStructure` with `confinement=2` and strain enabled.

**Format:** 2D grid with header `# x(A) y(A) eps_xx eps_yy eps_zz eps_xy eps_xz eps_yz`, blank-line separators.

| Column | Meaning | Units |
|--------|---------|-------|
| 1 | x-coordinate | Angstrom |
| 2 | y-coordinate | Angstrom |
| 3 | eps_xx (normal strain, x) | dimensionless |
| 4 | eps_yy (normal strain, y) | dimensionless |
| 5 | eps_zz (normal strain, z) | dimensionless |
| 6 | eps_xy (shear strain, xy) | dimensionless (always 0 for plane-strain) |
| 7 | eps_xz (shear strain, xz) | dimensionless (always 0 for plane-strain) |
| 8 | eps_yz (shear strain, yz) | dimensionless |

**Interpretation:** Strain is computed via continuum elasticity (plane-strain formulation) relative to a reference lattice constant from the substrate (first material). The strain modifies the band edges via the Pikus-Bir Hamiltonian before the k-sweep begins. `eps_xy` and `eps_xz` are identically zero in the plane-strain formulation.

## 8. g-factor Calculation Profile (stdout)

In `gfactorCalculation` with QW mode (`confinement=1`), the band edge profile is written to Fortran unit 101 (stdout) rather than a file:

```fortran
write(101,*) cfg%z(i), profile(i,1), profile(i,2), profile(i,3)
```

This produces the same 4-column format as the QW `potential_profile.dat` but goes to standard output. This is a legacy path and may appear intermixed with other terminal output.
