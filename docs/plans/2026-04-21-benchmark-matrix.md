# Benchmark Matrix for Docs Physics Revamp

Purpose: track which lecture chapters are benchmarked, what observables matter,
what tolerance is acceptable, and whether the current code/docs state is
defensible.

Policy:
- This is a research-grade internal control document, not presentation prose.
- nextnano++ tutorials are the primary external framework where a close match
  exists.
- Literature or analytic limits are used where nextnano++ does not provide a
  direct tutorial match.
- If a row is `blocked` or `high-risk`, the corresponding lecture claims should
  be downgraded or removed until the benchmark closes.

Status key:
- `validated`: current lecture claims are broadly defensible; rerun later for provenance/regression only.
- `partial`: chapter has a usable benchmark anchor, but one or more figures or claims still need rerun or tightening.
- `blocked`: known figure/parser/unit problems prevent the current chapter from being trusted quantitatively.
- `high-risk`: no adequate external reproduction yet; strong claims should not survive without dedicated hardening.
